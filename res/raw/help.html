Introduction

Flightlog was build to manage model data. This helps to retsore data in case of a new radio, or just for history. You can store model data and flights for each model, only limited by memory.

Version Iínformation

Current verion is 1.11 build 06/2014.

Running on android platform 2.3.6 and above.

Tested on Sony Xperia X10 and Tipo, Samsung Galaxy Ace, S3, S4 (up to Android 4.3).

On Android 4.x.x the new feature actionbar is used, the preferences, the speechtest and connect buttons are rendered in the title.

Please disable in 'Developer Options' the purge activity everey time checkbox, since this will restart the whole app when launching e.g. the preferences. It is only user selectable on some Android 4.0.4 devices.

Import and Export of model data

In menu options select export database, this will export three files in csv format which can be imported in any calc program such as Excel, OpenOffice and LibreOffice even editable in every texteditor.

Inspect your sdcard with a fileexplorer tool like Astro Dateimanager. The app will create a new directory under flight_log, which is named 'export'

The filename consists of a timestamp and a functional part which is model_data, flight_data and model_data_complete and the file extension 'csv'.

model_data contains only the model data each row one model. Take this file as a basis for later import.

flight_data contains all flights with a reference by name to the model data.

model_data_complete contains all data a row for the model data, with indentation as subsection the flightdata.

Set the correct delimter in options to ';', this is default and may be untouched.

Other progams need e.g. ','.

There is a minor bug in the excel import since version Excel 2003. The general RTF description for csv file format was the base for this export.

Excel handles the breaks within a cell not correct. Normally when you press ALT+ENTER in a cell this invokes a LINEFEED (HEX 0xA).

The delimiter for each row is a 0x0A as well so Excel will break and puts all data in a new row.

The Excel exports itself will fail, in case your reimport it into Excel again!

There is no workaround.

Other calculation programs such as OpenOffice calc treat the csv files in a correct way.

The import is only for importing model data, no flights can be imported at this time. May be in one of the next versions.

The base will be the model_data.csv file.

The file must contain a header and at least 14 fields. The naming of the headers doesn't care.

The 'Bild Pfad' (last field) will not be imported, so leave it blank.

If you like to import a field with more lines, add an ALT+ENTER at the end of the cell, which will invoke a carriage return.

This option will be treated within the import into the database.

The ID (first field) is just for information purpose, it will be generated during import, so just don't care.

Put the file into a new directory called 'import' (casesensitive) under flight_log and name it model_data.csv

Select import modeldata from options menu and wait.

The imported model names will be displayed. There are some error messages:

File not found

Incorrect field count

general error

In any case test the correct setup of the import file.

The file will be deleted after successful import.
